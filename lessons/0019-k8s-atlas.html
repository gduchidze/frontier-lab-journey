Spotlight 1 — Graceful shutdown for LLM engines

Termination sequence: pod marked Terminating → removed from Service endpoints → preStop hook runs → SIGTERM → grace period → SIGKILL. Default grace: 30 s. A 70B engine mid-generation on a long output needs minutes. The production pattern:

terminationGracePeriodSeconds: 300
lifecycle:
  preStop:
    exec:
      command: ["/bin/sh","-c","sleep 15"]  # let endpoint removal propagate
                                            # before the engine even sees SIGTERM

Plus a PDB so drains never take the fleet below N−1. Without these, every node upgrade = user-visible failures. This is a 10-line diff worth a paragraph in your design doc.

Spotlight 2 — Jobs are half of AI compute

Evals, batch scoring, embeddings backfills, distillation data generation — none of it wants a Deployment. Jobs give you completion tracking, retry with backoffLimit, parallelism fan-out, and (with Indexed completion mode) a per-pod index — the poor man's rank. Add Kueue when multiple teams' Jobs compete for the same GPUs. Your lab runs a real batch-inference Job against the stack.

Spotlight 3 — Operators: the pattern behind everything you installed

CRD = new noun in the API (ServiceMonitor, Certificate, Application). Operator = controller making that noun true, in a loop, forever. That's the whole trick — the same reconciliation you watched in lesson 1, pointed at domain logic. production-stack's operator/ directory manages engine fleets this way. You'll write a toy CRD in the lab to break the mystique; kubebuilder is the real tool when a team needs one.

Spotlight 4 — Init containers for weight pre-loading

Scaling lag (lesson 10's pain) is mostly weight-loading. Init-container pattern: a downloader image populates the PVC/hostPath cache BEFORE the engine container starts; combined with RWX storage or node-local caches, new replicas skip the download entirely. Streaming loaders (Run:ai model streamer, tensorizer — lesson 20) attack the same seconds.

Spotlight 5 — Helm vs Kustomize (the eternal interview question)

Helm: packaging + templating + lifecycle (installable product, values API — what you built). Kustomize: patch-based env overlays with zero templates (fleet configs, no chart to maintain). Real platforms: Helm for third-party software, Kustomize overlays (sometimes ON TOP of rendered Helm) for env-specific tweaks. Saying exactly that sentence settles the debate in most rooms.

Lab (labs/lab-19): touch four gaps

Batch inference Job

Job (parallelism 2, completions 6) whose pods curl the router with prompts from a ConfigMap, writing outputs to a shared emptyDir→log. Watch kubectl get jobs -w count completions. Solution: solutions/k8s-advanced/batch-job.yaml.

Graceful drain drill

Add preStop + 300 s grace + a PDB (minAvailable: 1) to the engine chart. Then kubectl drain llm-stack-worker --ignore-daemonsets --delete-emptydir-data mid-load: PDB paces the eviction, engine drains politely. kubectl uncordon after.

kubectl debug

kubectl debug -it <engine-pod> -n llm --image=busybox --target=vllm — poke /proc, see the engine's world from a sidecar that wasn't there a second ago.

Toy CRD

Apply a ModelEndpoint CRD + one CR (solution provided). kubectl get modelendpoints — your own noun in the API. No controller yet; discuss with the teacher what the reconcile loop WOULD do (that conversation is the operator interview).

Check yourself

Node upgrade killed generations mid-token despite a PDB. Missing piece?

PDB paces HOW MANY pods evict — not how gently each dies. In-flight work needs terminationGracePeriodSeconds + preStop drain.

Nightly eval over 10k prompts across 4 workers. Right primitive?

Run-to-completion + parallelism + per-pod index for sharding = Job; CronJob schedules it. Deployments never finish; StatefulSets solve identity, not completion.

What turns a CRD into an operator?

CRD alone is inert storage (your lab proves it). The operator IS the reconcile loop watching those objects and making reality match.
