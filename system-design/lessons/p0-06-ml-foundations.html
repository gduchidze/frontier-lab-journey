ML Foundations

Loss, gradient descent, generalization — and the metrics that decide whether a model ships.

Mission tie. Even in Anthropic's systems rounds, ML fluency is assumed: propose quantizing a model or routing traffic to a cheaper one, and the follow-up is "how do you know quality didn't drop?" The only defensible answer is a held-out set, a metric, and a threshold — this lesson's exact content. Chip Huyen's Designing Machine Learning Systems (ch. 1–3) frames ML this way — as an engineering discipline with SLO-like targets, not a bag of algorithms. (Exponent's Anthropic guide)
Lab first (ADHD-friendly order) Don't read this top-to-bottom cold. Open labs/lab-p0-06-gradient-descent/, run python3 gradient_descent_lab.py, skim the three output tables, then read — each section below explains a number you have already seen.

1 · Learning is curve-fitting with a report card

A model is a parameterized function f(x; w); training is searching for parameters w that make it match data. In supervised learning you have labeled pairs (x, y) — regression predicts numbers, classification predicts categories. Unsupervised learning finds structure without labels (clustering, dimensionality reduction). LLM pretraining is self-supervised: the label is just the next token, so the labels come free with the text. (Google ML Crash Course, DMLS ch. 1)

The report card has two grades: error on data you trained on, and error on data you didn't. Only the second one ships.

2 · The loss is the spec

The loss function turns "match the data" into a single number to minimize. Choosing it is choosing what the model optimizes for — it is the SLO of training. (MML, ch. 8)

TaskLossReads as
RegressionMSE (mean squared error)average squared miss — big misses punished quadratically
ClassificationCross-entropynegative log of the probability given to the right answer
LLM pretrainingCross-entropy over next tokenssame loss; its exponential is perplexity (lesson p0-14)
Vocabulary Loss is what training minimizes; a metric (accuracy, precision, recall) is what humans judge by. They are usually different — cross-entropy is differentiable, accuracy is not — and confusing the two is a classic review-round mistake.

3 · Gradient descent, one step at a time

The gradient is the direction of steepest loss increase; step the opposite way, scaled by the learning rate. That is the whole algorithm — everything from linear regression to Claude's pretraining is this loop at different scales. (MML ch. 7, 3Blue1Brown)

Data: (x, y) = (1, 2) and (2, 4).   Model: ŷ = a·x, start a = 0, lr = 0.1.
Loss: MSE(a) = mean((a·x − y)²)        grad = mean(2·(a·x − y)·x) = 5(a − 2)

step 0: preds (0, 0)  errors (−2, −4)  MSE = (4+16)/2 = 10.0
        grad = −10        a ← 0 − 0.1·(−10) = 1.0
step 1: preds (1, 2)  errors (−1, −2)  MSE = 2.5
        grad = −5         a ← 1.0 + 0.5 = 1.5
step 2: grad = −2.5       a ← 1.75         MSE = 0.625
Each step multiplies the distance to a = 2 by (1 − lr·5) = 0.5: geometric convergence.

In practice you use SGD: the gradient is estimated on a random mini-batch, not the full dataset — noisier per step, vastly cheaper per step.

4 · The learning-rate knob

Reuse the factor from the worked example: each update multiplies the error in a by (1 − lr·curvature). Three regimes fall out:

  • Too small — factor ≈ 1: converges, but wastes your step budget (lab: lr=0.001 barely moves in 200 steps).
  • Right — factor well inside (−1, 1): fast geometric convergence.
  • Too big — factor beyond −1: each step overshoots further than it started; with lr = 0.45 the same problem gives a → 0, 4.5, −1.1, 5.9… divergence. The lab's sweep shows loss hitting 10¹².

This is the single highest-leverage hyperparameter, and the loss curve diagnoses it: flat = too small, exploding = too big, smooth decay = right. Momentum and Adam (lesson p0-07) are refinements of this knob, not replacements. (Karpathy, "A Recipe for Training Neural Networks")

5 · Train / validation / test discipline

Three splits, three jobs, three touch frequencies. Blur them and your numbers stop meaning anything. (scikit-learn: cross-validation)

SplitUsed forTouched
Trainfitting parameters wevery gradient step
Validationchoosing hyperparameters, early stopping, model selectionevery experiment
Testthe final, reportable estimate of real-world erroronce, at the end

Leakage is any path by which information about held-out examples reaches training: duplicates across splits, normalizing with statistics computed on the full dataset, or splitting time-series randomly instead of by time. Leakage makes offline numbers great and production numbers a surprise. (DMLS ch. 4 on leakage)

6 · Bias–variance, and the fixes

Two error signatures, two opposite treatments. Diagnose from the train/val gap before reaching for a fix:

Train errorVal errorDiagnosisFix
highhighhigh bias (underfit)bigger model, better features, train longer
lowhighhigh variance (overfit)more data, regularization, simpler model
lowlowdoneship it; spend the budget elsewhere

Regularization buys variance reduction without collecting data. L2 adds λ·Σw² to the loss, shrinking weights toward zero and smoothing the learned function; early stopping halts before the model memorizes noise; dropout (p0-07) randomly disables units. In the lab, a degree-9 polynomial on 10 points hits val MSE ≈ 10⁸ — and a whisper of L2 (λ = 0.001) drops it back to ≈ 0.1. Train error always improves with capacity; only held-out error is U-shaped.

7 · Precision, recall, AUC

Classifiers output scores; a threshold turns scores into decisions, producing true/false positives and negatives. Precision = TP/(TP+FP): when it flags, is it right? Recall = TP/(TP+FN): of the real positives, how many did it catch? Sliding the threshold trades one for the other — the lab table shows precision 0.52→1.00 while recall falls 1.00→0.05. F1 is their harmonic mean; AUC is threshold-free: the probability a random positive outranks a random negative, 0.5 = coin flip. (scikit-learn: model evaluation)

The accuracy trap With 99% negatives, "predict negative always" scores 99% accuracy and catches nothing. On imbalanced data, quote precision/recall at your operating threshold — never bare accuracy.

8 · Sharp edges

  • Test-set overfitting without touching it. Run 50 experiments, pick the best by validation, and validation itself is now optimistically biased — teams "climb the leaderboard" into models that fail in production.
  • Leakage via preprocessing. Fit scalers, vocabularies, and feature statistics on the train split only, then apply to the rest.
  • Metric–cost mismatch. Optimizing F1 when false positives cost 10× false negatives ships the wrong threshold. Price the errors first (lab E4).
  • Distribution shift. Offline metrics are computed on yesterday's distribution; production serves today's. Monitoring is the fix, not a bigger test set. (DMLS ch. 8)
  • Noisy comparisons. Model A beating model B by 0.3% on 500 examples is likely noise — significance testing arrives in p0-13.

10 · Project step

Our spine project — the LLM inference platform — will eventually route each request to a small cheap model or a large expensive one via a classifier ("is this query hard?"). Write its 5-line eval sheet: (1) which error is worse, sending hard queries to the small model (quality hit) or easy ones to the big model (cost hit)? (2) therefore, precision or recall priority, and a threshold; (3) where the held-out set comes from (real logged queries, deduplicated); (4) when the test set gets spent; (5) the minimum win that justifies shipping. The same discipline prices quantization and speculative decoding quality checks in Phase 4, and cross-entropy returns as perplexity in p0-14.

Interview talking points

  • "Training minimizes loss on the train set, but the product ships generalization — the train/validation gap is the first thing I diagnose."
  • "Both errors high means bias — bigger model, better features. Big gap means variance — more data, regularization, simpler model."
  • "Learning rate is the highest-leverage hyperparameter: the loss curve tells you which side you're on — flat is too small, exploding is too big."
  • "Validation is for making choices; test is a one-shot final answer. Every peek at test leaks it into model selection."
  • "Accuracy is meaningless on imbalanced data — I quote precision and recall at the operating threshold, and AUC for threshold-free ranking."
  • "The threshold itself is a product decision — the relative cost of false positives versus false negatives — not an ML decision."
