Manifest Anatomy

Annotated skeletons of every resource kind this course uses. Canonical source: Kubernetes API reference.

Every manifest, always

apiVersion: apps/v1        # API group/version the kind lives in
kind: Deployment           # what you're declaring
metadata:
  name: vllm               # unique within namespace
  namespace: llm
  labels: { app: vllm }    # tags for selectors and humans
spec:                      # desired state — shape depends on kind
  ...

Deployment (full production shape)

apiVersion: apps/v1
kind: Deployment
metadata:
  name: vllm
  namespace: llm
spec:
  replicas: 2
  selector:                          # MUST match template labels
    matchLabels: { app: vllm }
  template:                          # the Pod recipe
    metadata:
      labels: { app: vllm }         # what Services select on
    spec:
      containers:
      - name: vllm
        image: vllm-cpu:local
        imagePullPolicy: IfNotPresent   # required for kind-loaded images
        args: ["--model", "Qwen/Qwen3.5-0.8B"]
        env:
        - name: VLLM_CPU_KVCACHE_SPACE
          value: "4"
        - name: HF_TOKEN                 # secret injection pattern
          valueFrom:
            secretKeyRef: { name: hf-token, key: HF_TOKEN }
        ports:
        - containerPort: 8000
          name: http                     # NAME ports — ServiceMonitors need it
        resources:
          requests: { cpu: "4", memory: 6Gi }   # scheduler reservation
          limits:   { cpu: "6", memory: 8Gi }   # kill ceiling (mem → OOMKilled)
        startupProbe:                    # boot grace: 60 × 10s = 10 min
          httpGet: { path: /health, port: 8000 }
          periodSeconds: 10
          failureThreshold: 60
        readinessProbe:                  # traffic gate
          httpGet: { path: /health, port: 8000 }
          periodSeconds: 10
        livenessProbe:                   # hang detector → restart
          httpGet: { path: /health, port: 8000 }
          periodSeconds: 20
          failureThreshold: 3
        volumeMounts:
        - { name: hf-cache, mountPath: /root/.cache/huggingface }
        - { name: dshm,     mountPath: /dev/shm }
      volumes:
      - name: hf-cache
        persistentVolumeClaim: { claimName: hf-cache }
      - name: dshm                       # PyTorch shared memory
        emptyDir: { medium: Memory, sizeLimit: 2Gi }

Service

apiVersion: v1
kind: Service
metadata:
  name: vllm
  namespace: llm
spec:
  # type: ClusterIP (default) | NodePort | LoadBalancer
  selector: { app: vllm }     # routes to Ready pods with this label
  ports:
  - name: http                # named → ServiceMonitor can reference
    port: 8000                # the Service's own port
    targetPort: 8000          # container port (number or port name)
    # nodePort: 30080         # NodePort type only, range 30000-32767

In-cluster DNS: <name>.<namespace>.svc.cluster.local → e.g. vllm-router.llm.svc.cluster.local.

PersistentVolumeClaim

apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: hf-cache
  namespace: llm
spec:
  accessModes: [ReadWriteOnce]   # RWO = one node at a time (matters at replicas>1!)
  resources:
    requests:
      storage: 10Gi
  # storageClassName: standard   # kind's default provisioner if omitted

ConfigMap and Secret

apiVersion: v1
kind: ConfigMap
metadata: { name: app-config, namespace: llm }
data:
  LOG_LEVEL: "info"
  settings.yaml: |
    key: value
---
apiVersion: v1
kind: Secret
metadata: { name: hf-token, namespace: llm }
type: Opaque
stringData:                  # plaintext here; stored base64. NEVER commit real tokens
  HF_TOKEN: "hf_xxx"

Create secrets imperatively instead of committing them: kubectl create secret generic hf-token --from-literal=HF_TOKEN="$HF_TOKEN" -n llm.

HorizontalPodAutoscaler

apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata: { name: vllm, namespace: llm }
spec:
  scaleTargetRef: { apiVersion: apps/v1, kind: Deployment, name: vllm }
  minReplicas: 1
  maxReplicas: 2
  metrics:
  - type: Resource
    resource:
      name: cpu
      target: { type: Utilization, averageUtilization: 70 }
  behavior:
    scaleDown:
      stabilizationWindowSeconds: 300   # don't kill engines on brief lulls

ServiceMonitor (Prometheus Operator CRD)

apiVersion: monitoring.coreos.com/v1
kind: ServiceMonitor
metadata: { name: vllm, namespace: llm }
spec:
  selector:
    matchLabels: { app: vllm }   # matches SERVICE labels, not pod labels
  endpoints:
  - port: http                   # SERVICE PORT NAME — #1 gotcha
    path: /metrics
    interval: 15s

RBAC trio (used by the router)

apiVersion: v1
kind: ServiceAccount
metadata: { name: vllm-router, namespace: llm }
---
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata: { name: pod-reader, namespace: llm }
rules:
- apiGroups: [""]
  resources: ["pods"]
  verbs: ["get", "list", "watch"]
---
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata: { name: vllm-router-pod-reader, namespace: llm }
subjects:
- { kind: ServiceAccount, name: vllm-router, namespace: llm }
roleRef:
  { kind: Role, name: pod-reader, apiGroup: rbac.authorization.k8s.io }
