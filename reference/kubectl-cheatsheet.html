kubectl Cheat Sheet

The commands this course actually uses. Canonical source: kubectl quick reference.

Inspect

CommandWhat
kubectl get nodesCluster machines and their status
kubectl get pods -AAll pods, all namespaces (-A = everywhere)
kubectl get pods -o widePods + which node, IP
kubectl get pods -wWatch live changes (Ctrl+C exits)
kubectl get deployment,svc,pods -l app=XEverything matching label
kubectl describe pod NAMEFull detail + Events — first stop when broken
kubectl logs NAMEContainer stdout (-f follows, -l app=X by label)
kubectl cluster-infoAPI server address

Create / change

CommandWhat
kubectl apply -f file.yamlDeclare desired state from YAML (the production way)
kubectl create deployment N --image=I --replicas=2Quick generator (learning/dev only)
kubectl expose deployment N --port=80Generate Service for a Deployment
kubectl scale deployment N --replicas=3Change replica count
kubectl rollout status deployment/NWait for rollout to finish
kubectl delete -f file.yamlDelete what the YAML declared

Debug / access

CommandWhat
kubectl port-forward svc/N 8080:80Tunnel local port → Service
kubectl exec -it POD -- shShell inside a container
kubectl get events --sort-by=.lastTimestampRecent cluster events, newest last

OOMKilledExceeded memory limitRaise limit or shrink workload
Running but READY 0/1Readiness probe failingdescribe → probe config

kind (local clusters)

CommandWhat
kind create cluster --config k8s/kind-config.yamlCreate from config
kind get clustersList clusters
kind load docker-image IMG --name llm-stackPush a local image into the cluster (no registry needed)
kind delete cluster --name llm-stackDestroy

Glossary (course-wide)

TermDefinition
ClusterControl plane + nodes acting as one computer
Control planeAPI server, scheduler, etcd — decides, stores, assigns
NodeMachine (or kind container) that runs Pods
PodSmallest deployable unit; mortal; 1+ containers
DeploymentKeeps N replicas of a Pod template alive (reconciliation)
ServiceStable address load-balancing over Pods by label selector
NamespaceNamed partition of cluster resources
Label / selectorKey-value tags; how Services and controllers find Pods
ManifestYAML file declaring desired state
ReconciliationLoop driving actual state toward desired state
