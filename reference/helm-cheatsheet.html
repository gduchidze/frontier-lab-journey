Helm Cheat Sheet

Commands, chart layout, and the template syntax survival kit. Canonical: helm.sh/docs.

Commands

CommandWhat
helm install NAME ./chartRender + apply as release NAME
helm install NAME repo/chart -f values.yamlInstall from repo with value overrides
helm upgrade NAME ./chart --set a.b=cApply changes (new revision)
helm rollback NAME [REV]Revert to previous / given revision
helm history NAMERevision log
helm list -AAll releases, all namespaces
helm uninstall NAMEDelete release resources
helm template ./chartRender locally, no cluster — debugging #1
helm lint ./chartStatic checks
helm get values NAMEValues a release was installed with
helm repo add NAME URL && helm repo updateRegister chart repo

Chart layout

charts/slm-stack/
├── Chart.yaml          # name, version (chart), appVersion (software)
├── values.yaml         # default knobs — the chart's public API
├── templates/          # manifests with {{ }} holes
│   ├── _helpers.tpl    # named template snippets (define/include)
│   ├── vllm-deployment.yaml
│   ├── vllm-service.yaml
│   └── vllm-pvc.yaml
└── charts/             # subchart dependencies

Template syntax survival kit

{{ .Values.vllm.model.name }}            # value lookup
{{ .Values.tag | default "latest" }}     # pipe through function
{{ .Values.model.name | quote }}         # always quote strings you don't control
{{ .Release.Name }} {{ .Release.Namespace }}   # install-time context
{{ .Chart.Name }} {{ .Chart.Version }}

# whole-block YAML insertion — indentation is THE classic bug:
resources:
  {{- toYaml .Values.vllm.resources | nindent 10 }}
#  ^ "-" trims preceding whitespace/newline; nindent N re-indents block to N

# conditionals / loops
{{- if .Values.dashboard.enabled }} ... {{- end }}
{{- range $key, $val := .Values.env }}
- name: {{ $key }}
  value: {{ $val | quote }}
{{- end }}

Values precedence (low → high)

  1. values.yaml in the chart
  2. -f my-values.yaml (last file wins if several)
  3. --set key=value

Debug ritual

helm lint ./chart                        # 1. static errors
helm template ./chart | less             # 2. is generated YAML right?
helm template ./chart | kubectl apply --dry-run=server -f -   # 3. would API accept it?
helm upgrade NAME ./chart --debug        # 4. verbose install
